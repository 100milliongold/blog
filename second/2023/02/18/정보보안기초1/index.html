정보​

    • 문제점 -> 해결책을 제시 하는 문제

    • 법률 : 데이터 상법, 개인정보 보호법, 정보통신망법, 신용정보 관리법

      • 규제 , 근거 가되는 기준이 법에 의존하기 때문에 알아야함
      • 표준이 됨 : ISO 27001 , ISMS-P

    • ISMS-P 인증기준 안내서 정독

    • 1페이지에 7분정도 시간이 소요됨

    • 장기적인 싸움이 됨 (마라톤)

  • 집에 가서 할일 5년치 기출문제를 분석해야 해야됨

    • 방향성을 잡고 선택과 집중을 할수 있도록 한다.

  • 1교시 문제일때 출제자가 기술사인지, 교수인인지 파악을 한다.

    • 2,3,4 교시에 같은 패턴으로 출제 가능성이 있음

  • 출근 때마다 토픽을 연습해야 한다.

  • 답안 제출시 오답 수정시 탈락이 가능성이 있음

정보보안의 개념​

  • 외부의 악의적인 해킹이나 내부의 불법적 정보 유출시도로 부터 조직과 고객의 정보자산을 보호하기 위한 관리적, 물리적, 기술적 대응 체계

    • 정의 연습
      • 5W 1H : ~ 목적 ~ 기술을 이용 ~ 종결명사 (PK)
      • X ~ 하는 것 문장은 쓰면안된다 : 점수 안줌

  • 위험을 수용 가능한 수준으로 감소시키는 일련의 조치 Risk = (Asset) XXX (vulnerability) XXX (Threat) 위험 = 영향도 * 확룰 * 취약

    • 리스크는 자산이라고도 한다. 영향도는 취약점
구분설명핵심요소
기밀성 (Confidentiality)내외로 부터 불법적 정보 유츨의 방지 해당 정보에 대한 권한이 부여된 자들만이 접근가능하도록 보장암호화, 접근 통제
무결성 (Integrity)정보의 위변조 및 파괴를 예방하고 방지 원천 데이터나 사용자가 원하는 정보나 시스템에 적시에 접근 보장
서버에 로그를 적재시 위변조 방지를 위하여 별도의 저장소 또는 헤시값을 부여하여 무결성을 보장한다.		보안관제, 이중화
가용성 (Availability)해킹으로 안한 시스템 동작 불능 예방 인가된 사용자가 원하는 정보나 시스템이 적시에 접근 보장보안관제, 이중화
인증 (Authentication)정보나 시스템에 접근하는 사용자가 정당함을 확인 알고 있는 것, 가지고 있는것 , 신체의 일부(지문)PKI, 생체인증
부인방지 (Non-Repudiation)행위를 부인하는 것을 봉쇄전자 서명

  • 사례를 적시하면 좋음

  • 기업과 국가의 지속가능 경영을 위해서는 정보보호는 필수이며, 정보공학기술사의 가장 중요한 임무임

CC 인증​

  1. ISO 15408 CC인증의 개념

    • 국가마다 서로 다른 정보보호시스템 평가 기준을 연동하고 평가결과를 상호 인증하기 위해 제정된 평가기준
    • 국제표준으로 1999년 6월 승인됨

  2. 목적

    • 정보보호 시스템 보안등급 평가에 신뢰성 부여
    • 현존하는 쳥가기준과 조화를 통해 평가결과의 상호 인정
    • 정보보호 시스템의 수출입에 소요되는 인증 비용 절감으로 국제 유통 추진

  3. CC의 구조

    1부. 소개및 일반모델- IT 보안성 평가의 원칙과 개념 정의
    2부. 보안 기능 요구사항
    3부. 보증 요구사항

  4. CC 인증의 핵심 PR 와 ST

    구분PPST
    개념
    목적
    독립성
    적용성
    관계성
    완전성
    예시
  • PP,ST, TOE 에 대한 단어가 꼭 들어 가야 한다.

  1. CC인증 과정

    단계명칭예시
    EAL1- 기능적 시험- 기능 명세서, 설계서
    EAL2- 구조적 시험- 기본설계서, 기능시험서, 취약점분석서
    EAL3- 방법론적 시험, 점검- 생명주기 지원, 개발 보안 , 오용 분석서
    EAL4- 방법론적 설계, 시험, 점검- 상세설게서, 보안정책서, 일부 소스코드, 상세시험서
    EAL5- 준정형적 설계, 시험- 보안 기능 / 전체 소스코드. 구조 명세서, 은닉채널 분석서
    EAL6- 준정형적 검증된 설계, 시험- 보안 기능 / 전체 소스코드. 구조 명세서, 은닉채널 분석서
    EAL7- 정형적 검증- 보안 기능 / 전체 소스코드. 구조 명세서, 은닉채널 분석서

ISO 27001​

  1. ISO/IEC 27001 정의

    • 국제표준화 기구에서 정보보호관리를 위해 제장한 표준화된 실무규약

  2. 정보보호관리체계(ISMS-P) 국제표준의 필요성

    • 정보보호대책 강화 : 크고 복잡한 조직에 대한 쳬계적인 접근방법 미비
